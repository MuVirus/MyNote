# 基础
## 1. 指针 vs 引用
### 面试问题1：说说指针和引用的区别
#### 标准答案
1. 本质：指针是变量（存储地址），引用是变量的别名；
2. 空值：指针可以为空（nullptr），引用必须绑定有效对象，不能为NULL；
3. 修改：指针可以重新指向其他对象，引用一旦绑定不可更改；
4. 语法：指针需要解引用`（*）`，引用直接使用；
5. 内存：指针占内存（32位4字节/64位8字节），引用不占额外内存；
6. 多级：支持多级指针（int**），无多级引用。
#### 核心要点
- 引用本质是编译器层面的语法糖，底层仍用指针实现；
- 函数参数优先用const引用（避免拷贝+防止修改）。
### 面试问题2：引用到底占不占用内存
- 在 C++ 语言标准层面（逻辑上）： 引用不占用额外的内存空间，它只是一个别名（Alias）。
- 在编译器实现层面（物理上）： 在大多数常见场景下，引用确实占用内存（通常实现为指针），但在编译器优化后（如内联、寄存器分配），它往往不占用实际内存。

### 面试问题2：const与constexpr的区别
#### 标准答案
`const`
- 用于定义变量为只读，在运行时只初始化一次。
- 常见用法：修饰局部变量、常量对象方法、函数参数（避免拷贝）。
`constexpr`
- C++11 引入，要求变量或表达式在 编译期间计算，如代码中的常量折叠。
- 常见用法：声明常量值、constexpr 函数 提高运行效率。

## 3. static
### 面试问题：static的作用
#### 标准答案：
1. 修饰全局变量：作用域限制在当前文件，避免多文件命名冲突；
2. 修饰局部变量：生命周期延长至程序结束，初始化仅一次；
3. 修饰类成员变量：所有对象共享，不占对象内存，需类外初始化；
4. 修饰类成员函数：无this指针，只能访问静态成员变量/函数。
#### 核心要点：
- 静态成员函数不能是虚函数；
- 局部static线程不安全（C++11之前，初始化过程非原子化，且标准未要求编译器做线程安全保障，多线程并发初始化时会出现竞态条件；C++11仅修复了“初始化阶段”的线程安全，static变量常规读写仍需手动同步）。


## 4. inline
### 面试问题：inline函数的作用及与宏的区别
#### 标准答案：
1. 作用：编译器将函数体嵌入调用处，减少函数调用开销；
2. 与宏的区别：
   - 类型安全：inline有类型检查，宏无；
   - 编译处理：inline由编译器处理，宏由预处理器替换；
   - 复杂逻辑：inline支持复杂函数，宏易出错（如宏定义#define ADD(a,b) a+b → ADD(1+2,3) 结果不符合预期）。
#### 核心要点：
- inline是建议，编译器可拒绝（如函数体过大）；
- 类内定义的成员函数默认inline。
