# 基础
## 1. 指针 vs 引用
### 面试问题1：说说指针和引用的区别
#### 标准答案
1. 本质：指针是变量（存储地址），引用是变量的别名；
2. 空值：指针可以为空（nullptr），引用必须绑定有效对象，不能为NULL；
3. 修改：指针可以重新指向其他对象，引用一旦绑定不可更改；
4. 语法：指针需要解引用`（*）`，引用直接使用；
5. 内存：指针占内存（32位4字节/64位8字节），引用不占额外内存；
6. 多级：支持多级指针（int**），无多级引用。
#### 核心要点
- 引用本质是编译器层面的语法糖，底层仍用指针实现；
- 函数参数优先用const引用（避免拷贝+防止修改）。
### 面试问题2：引用到底占不占用内存
- 在 C++ 语言标准层面（逻辑上）： 引用不占用额外的内存空间，它只是一个别名（Alias）。
- 在编译器实现层面（物理上）： 在大多数常见场景下，引用确实占用内存（通常实现为指针），但在编译器优化后（如内联、寄存器分配），它往往不占用实际内存。

### 面试问题2：const与constexpr的区别
#### 标准答案
`const`
- 用于定义变量为只读，在运行时只初始化一次。
- 常见用法：修饰局部变量、常量对象方法、函数参数（避免修改）。
`constexpr`
- C++11 引入，要求变量或表达式在 编译期间计算，如代码中的常量折叠。
- 常见用法：声明常量值、constexpr 函数 提高运行效率。

## 3. static
### 面试问题：static的作用
#### 标准答案：
1. 修饰全局变量：作用域限制在当前文件，避免多文件命名冲突；
2. 修饰局部变量：生命周期延长至程序结束，初始化仅一次；
3. 修饰类成员变量：所有对象共享，不占对象内存，需类外初始化；
4. 修饰类成员函数：无this指针，只能访问静态成员变量/函数。
#### 核心要点：
- 静态成员函数不能是虚函数；
- 局部static线程不安全（C++11之前，初始化过程非原子化，且标准未要求编译器做线程安全保障，多线程并发初始化时会出现竞态条件；C++11仅修复了“初始化阶段”的线程安全，static变量常规读写仍需手动同步）。


## 4. inline
### 面试问题：inline函数的作用及与宏的区别
#### 标准答案：
1. 作用：编译器将函数体嵌入调用处，减少函数调用开销；
2. 与宏的区别：
   - 类型安全：inline有类型检查，宏无；
   - 编译处理：inline由编译器处理，宏由预处理器替换；
   - 复杂逻辑：inline支持复杂函数，宏易出错（如宏定义#define ADD(a,b) a+b → 
   - ADD(1+2,3) 结果不符合预期）。
#### 核心要点：
- inline是建议，编译器可拒绝（如函数体过大）；
- 类内定义的成员函数默认inline。

## 5. 四大内存分区
### 面试问题：C++的四大内存分区分别是什么？各自特点和用途？
#### 标准答案
1. 栈区（Stack）：
   - 分配：编译器自动分配和释放（如函数参数、局部变量、函数返回地址）；
   - 特点：空间小（通常几MB）、访问速度极快、遵循先进后出（FILO）原则；
   - 风险：递归过深、局部数组过大易导致栈溢出，直接引发程序崩溃。
2. 堆区（Heap）：
   - 分配：程序员通过 `new/malloc` 主动申请，`delete/free` 手动释放；
   - 特点：空间大（受物理内存限制）、访问速度慢（需操作系统寻址）、存储无序；
   - 风险：未释放会造成内存泄漏，多次申请/释放易产生内存碎片。
3. 全局/静态区（Data Segment）：
   - 分配：存放全局变量、static 静态变量，程序启动时分配，退出时释放；
   - 细分：初始化区（已初始化的全局/静态变量）、未初始化区（BSS 段，默认初始化为 0）；
   - 特点：生命周期贯穿整个程序运行期，占用固定内存。
4. 代码区（Code Segment）：
   - 分配：存放程序二进制可执行指令、字符串常量、const 只读变量；
   - 特点：只读属性（防止篡改指令）、可共享（多个进程可复用同一份代码）；
   - 注意：修改代码区数据（如字符串常量）会触发程序崩溃。
#### 核心要点
- 堆区地址通常向高地址增长，栈区向低地址增长；
- 类的成员函数代码存于代码区，对象实例（非静态成员）：局部对象在栈区，new 创建的对象在堆区；
- 内存泄漏仅针对堆区，栈区/全局区由系统管理无泄漏风险。

## 6. 预处理、编译、汇编、链接的流程
### 面试问题：简述C++程序从源码到可执行文件的编译链接完整流程
#### 标准答案
1. 预处理（Preprocessing）：
   - 操作：预处理器处理源码（.cpp/.h），完成宏替换（#define）、头文件包含（#include）、条件编译（#ifdef/#if）、删除注释；
   - 输出：生成 `.i` 后缀的预处理文件（纯C++代码，无预处理指令）。
2. 编译（Compilation）：
   - 操作：编译器对 `.i` 文件进行语法分析、语义分析、优化，将高级语言转为汇编语言；
   - 输出：生成 `.s` 后缀的汇编文件。
3. 汇编（Assembly）：
   - 操作：汇编器将 `.s` 文件的汇编指令转为机器指令（二进制）；
   - 输出：生成 `.o/.obj` 后缀的目标文件（包含机器码、符号表、重定位信息）。
4. 链接（Linking）：
   - 操作：链接器将多个 `.o` 文件、系统库（如 libstdc++.so）合并，解析符号引用、重定位地址；
   - 细分：
     - 静态链接：将库代码直接嵌入可执行文件，生成独立可执行程序（体积大，无库依赖）；
     - 动态链接：仅记录库引用，运行时加载动态库（体积小，需依赖库文件）；
   - 输出：生成最终的可执行文件（Linux 下无后缀，Windows 下 .exe）。
#### 核心要点
- 预处理不做语法检查，编译阶段才会检测语法错误；
- 链接错误常见原因：符号未定义（如未链接对应库）、符号重复定义（如全局变量重复声明）；
- IDE 中的“编译”按钮实际是“预处理+编译+汇编+链接”的一站式操作。

## 7. 函数重载
### 面试问题：什么是函数重载？C++如何实现函数重载？
#### 标准答案
1. 定义：同一作用域内，函数名相同、参数列表（个数/类型/顺序）不同的多个函数，称为函数重载；
2. 实现原理：编译器通过“名字改编（Name Mangling）”区分重载函数，将函数名+参数类型组合为新名字（如 `add(int, int)` 改编为 `_Z3addii`，`add(double, double)` 改编为 `_Z3adddd`）；
3. 重载匹配规则：
   - 优先匹配参数完全一致的版本；
   - 若无完全匹配，尝试隐式类型转换匹配（如 int 转 double）；
   - 无匹配/匹配模糊时，编译报错。
#### 核心要点
- 返回值不同**不能**作为函数重载的依据（编译器无法通过返回值区分调用）；
- 函数重载仅发生在同一作用域（如不同命名空间的同名函数不算重载）；
- C 语言不支持重载（无名字改编机制），C++ 中可通过 `extern "C"` 禁用重载。

### 8. 默认参数
### 面试问题：C++默认参数的使用规则和注意事项
#### 标准答案
1. 定义：函数声明/定义时，为参数指定默认值，调用时可省略该参数（使用默认值）；
2. 使用规则：
   - 右到左原则：默认参数必须从参数列表最右侧开始连续指定（如 `void func(int a, int b=10, int c=20)` 合法，`void func(int a=10, int b)` 非法）；
   - 声明与定义一致：默认参数若在声明中指定，定义中不可重复指定（可省略）；
   - 调用规则：省略参数时需按顺序省略，不可跳过中间参数（如 `func(1)` 用默认 b/c，`func(1,,3)` 非法）。
3. 常见场景：简化函数调用，减少重载函数数量（如兼容不同参数个数的调用）。
#### 核心要点
- 默认参数值必须是编译期常量（如 `10` 合法，`变量` 非法，C++11 后可支持 constexpr 变量）；
- 默认参数与函数重载结合时，需避免二义性（如 `func(int a=10)` 和 `func()` 同时存在，调用 `func()` 会编译报错）；
- 类的成员函数中，默认参数在编译期确定，而非运行期（与虚函数动态绑定不同）。

### 9. 可变参数
### 面试问题：C++中可变参数的实现方式和使用注意事项
#### 标准答案
1. 传统C风格可变参数（stdarg.h）：
   - 实现：通过 `va_list/va_start/va_arg/va_end` 宏实现，适用于参数类型相同/可统一处理的场景；
   - 示例：`void print(int count, ...) { va_list args; va_start(args, count); ... va_end(args); }`；
   - 缺点：无类型安全检查、需手动指定参数个数/结束标志、不支持非POD类型（如自定义类）。
2. C++11 可变参数模板（Variadic Templates）：
   - 实现：通过模板参数包（`typename... Args`）和函数参数包（`Args... args`）实现，支持任意类型、任意个数的参数；
   - 示例：`template <typename... Args> void print(Args&&... args) { /* 递归展开参数包 */ }`；
   - 优点：类型安全、支持任意类型、编译期检查，是现代C++推荐方式。
3. C++17 fold表达式：简化可变参数模板的参数包展开（如 `(cout << ... << args)` 一键展开）。
#### 核心要点
- 传统可变参数需注意：`va_start` 第二个参数必须是最后一个确定参数，`va_arg` 需指定正确类型，否则会内存越界；
- 可变参数模板的参数包展开通常通过递归函数/逗号表达式/fold表达式实现；
- 标准库中的 `std::printf`（C风格）、`std::format`（C++20，类型安全）是可变参数的典型应用。
